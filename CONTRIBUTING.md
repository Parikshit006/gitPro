# Contributing to GitPro

## Project Philosophy
- Strict domain boundaries.
- Evidence before AI.
- Enterprise-first security and maintainability.

## Conventional Commits
We adhere to Conventional Commits (e.g., `feat: add metric`, `fix: resolve bug`).

## Branch Naming Convention
- `feat/<short-description>`
- `fix/<short-description>`
- `chore/<short-description>`
- `docs/<short-description>`

## Pull Request Rules
- PRs must describe the *Why* alongside the *What*.
- CI checks must pass prior to review.
- Requires at least one approving review before merge.

## Code Review Expectations
- Reviewers focus on architectural integrity, domain boundaries, and security.

**Note:** Until a branching strategy is introduced, all contributions target the `main` branch.
